# Escribir aplicaciones MFC
[!INCLUDE[vs2017banner](../assembler/inline/includes/vs2017banner.md)]

En este artículo se explica los pasos iniciales que toma para desarrollar la aplicación.  Primero, debe decidir qué tipo de aplicación que está escribiendo.  Algunas de las opciones se descritas en [Opciones de diseño de aplicaciones](../mfc/application-design-choices.md).  La aplicación será:  
  
-   ¿Ejecutándose en internet o en una intranet?  
  
-   ¿Ejecutándose en un cliente o en un servidor?  
  
-   ¿Ejecutándose en un explorador o como aplicación independiente?  
  
-   ¿Usar tecnología COM o activos?  
  
-   ¿Datos de transferencia mediante WinInet o monikeres asincrónicos?  
  
 Las decisiones determinan qué clases son adecuadas para la aplicación.  Las respuestas también ayudan a determinar las selecciones que se crea al ejecutar el Asistente para aplicaciones empiece a construye la aplicación.  
  
 Después de haber obtenido las decisiones de diseño iniciales sobre la aplicación de internet, puede utilizar el Asistente para aplicaciones para empezar.  Utilice el Asistente para aplicaciones para crear una aplicación esqueleto y modificar el código como se describe en los siguientes casos:  
  
-   Para un control ActiveX, vea [Controles ActiveX en internet](../mfc/activex-controls-on-the-internet.md).  
  
-   Para un documento activo, vea [Documentos activos en internet](../mfc/active-documents-on-the-internet.md).  
  
 Los siguientes artículos también proporcionan instrucciones para ayudarle inician las tareas de programación:  
  
-   [Opciones de diseño de aplicaciones](../mfc/application-design-choices.md)  
  
-   [Monikeres asincrónicos en internet](../mfc/asynchronous-monikers-on-the-internet.md)  
  
-   [Fundamentos de WinInet](../mfc/wininet-basics.md)
