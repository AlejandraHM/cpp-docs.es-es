# Compilar en el marco
[!INCLUDE[vs2017banner](../assembler/inline/includes/vs2017banner.md)]

El rol en la configuración de una aplicación con el marco de trabajo de MFC es proporcionar el código fuente de aplicación concreto y conectar componentes definir qué mensajes y comandos a los que responden.  Use las técnicas de C\+\+ de idioma y del estándar de C\+\+ para derivar dispone de clases específicas de la aplicación de los proporcionados por la biblioteca de clases y para reemplazar y aumentar el comportamiento de la clase base.  
  
 En temas relacionados, las tablas siguientes describen la secuencia general de operaciones que realizará normalmente y las responsabilidades con las responsabilidades de marco de trabajo:  
  
-   [Secuencia para compilar una aplicación con el marco](../mfc/sequence-of-operations-for-building-mfc-applications.md)  
  
-   [Secuencia de operaciones para crear aplicaciones OLE](../mfc/sequence-of-operations-for-creating-ole-applications.md)  
  
-   [Secuencia de operaciones para crear controles ActiveX](../mfc/sequence-of-operations-for-creating-activex-controls.md)  
  
-   [Secuencia de operaciones para crear aplicaciones de base de datos](../mfc/sequence-of-operations-for-creating-database-applications.md)  
  
 En general, puede seguir estas tablas como una secuencia de pasos para crear una aplicación MFC, aunque algunos pasos son opciones alternativas.  Por ejemplo, el tipo de uso una de las aplicaciones de clase de vista de varios tipos disponibles.
