# <a name="remote-automation"></a>Automatización remota
> [!NOTE]
>  Automatización remota es una tecnología obsoleta y no debe usarse para las aplicaciones nuevas. Visual C++ no es compatible con Windows 95. Se describen los casos en que se necesita admitir la automatización remota en [donde remoto automatización caber en](where-does-remote-automation-fit-in-q.md).  
  
 Automatización remota es un tipo de [automatización](../mfc/automation.md) que permite que un consumidor de interfaz a un proveedor de interfaz que reside en otro equipo, por ejemplo, se ejecuta en una red.  
  
 En este artículo se explica cómo crear objetos de automatización que se pueden llamar y ejecutar de forma remota y cómo crear controladores de automatización que pueden usar estos objetos de automatización remota. También se examinan las opciones de configuración y destaca las principales diferencias entre la automatización remota y DCOM (la versión distribuida de COM y OLE que permite a las interfaces diferentes de aquellas relacionadas con la automatización para que puedan llamar y ejecutar de forma remota).
