# CA0058
No se puede cargar los ensamblados a que se hace referencia.  
  
 Este error se produce si la herramienta de análisis no puede cargar los ensamblados a los que hace referencia el ensamblado analizado.  Como resultado, este error podría provocar CA0001 en otros lugares, porque en este caso podrían surgir muchos estados inesperados.  Si utiliza la herramienta de análisis de código administrado desde [!INCLUDE[vsprvs](../assembler/masm/includes/vsprvs_md.md)], a continuación se muestran algunos motivos que pueden provocar CA0058:  
  
-   Se reemplazó el ensamblado de entrada y ahora señala a un ensamblado que hace referencia a otros ensamblados situados fuera de la lista de referencias del archivo de proyecto de [!INCLUDE[vsprvs](../assembler/masm/includes/vsprvs_md.md)].  
  
-   Si se modifica un proyecto para llamar a un paso de compilación personalizada antes de que se ejecute FxCopCmd.exe y el proceso de compilación personalizada agrega más referencias al ensamblado de entrada, se genera el error CA0058.  
  
-   En C\+\+, es posible hacer referencia a los ensamblados mediante ForcedUsing en VCPROJ, reemplazando pasen o ForcedUsing desde la línea de comandos o agregando Some.dll \#using \<de\> origen únicamente.  
  
 En todos los casos, la solución para este problema es agregar la referencia que falta al propio proyecto de [!INCLUDE[vsprvs](../assembler/masm/includes/vsprvs_md.md)].
