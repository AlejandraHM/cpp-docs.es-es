# Plataformas Windows (CRT)
[!INCLUDE[vs2017banner](../assembler/inline/includes/vs2017banner.md)]

Las bibliotecas en tiempo de ejecución de C para compatibilidad [!INCLUDE[win8](../build/includes/win8_md.md)], [!INCLUDE[winserver8](../build/includes/winserver8_md.md)], [!INCLUDE[win7](../build/includes/win7_md.md)], [!INCLUDE[winsvr08](../build/includes/winsvr08_md.md)], Windows Vista, Service Pack 3 \(SP3\) de [!INCLUDE[winxp](../build/includes/winxp_md.md)] para x86, [!INCLUDE[winxp](../build/includes/winxp_md.md)] Service Pack 2 \(SP2\) para x64, y [!INCLUDE[WinXPSvr](../build/includes/winxpsvr_md.md)] Service Pack 2 \(SP2\) de [!INCLUDE[vs2012_upd01](../c-runtime-library/includes/vs2012_upd01_md.md)] para x86 y x64.  Todos estos sistemas operativos admiten la interfaz de programación \(API\) de la aplicación para Windows y proporcionan compatibilidad de Unicode.  Además, cualquier aplicación Win32 puede utilizar un juego de caracteres multibyte \(MBCS\).  
  
> [!NOTE]
>  [!INCLUDE[vs_dev11_long](../build/includes/vs_dev11_long_md.md)] no admite [!INCLUDE[winxp](../build/includes/winxp_md.md)] y el desarrollo de [!INCLUDE[WinXPSvr](../build/includes/winxpsvr_md.md)] .  Debe instalar [!INCLUDE[vs2012_upd01](../c-runtime-library/includes/vs2012_upd01_md.md)] para agregar compatibilidad para [!INCLUDE[winxp](../build/includes/winxp_md.md)] y [!INCLUDE[WinXPSvr](../build/includes/winxpsvr_md.md)].
